# MultiModalData

[textcaps](https://textvqa.org/textcaps/download/)



The table below details each of the requested Hugging Face datasets, including their respective domains, core data fields, and the explicit mapping of their variables to the inputs, intermediate projections, and output layers of the **MM-PEFT-ASD** architecture.

| Dataset Name & Hugging Face URL | Domain | Core Schema Fields | Conceptual Mapping to MM-PEFT-ASD Architecture |
| :--- | :--- | :--- | :--- |
| **Real Estate Search**<br>[Binaryy/multimodal-real-estate-search](https://huggingface.co/datasets/Binaryy/multimodal-real-estate-search) | Real Estate | <ul><li>`images` (List of property photos)</li><li>`description` (Text summary)</li><li>`location` (Text location details)</li><li>`details` (Structural/feature list)</li></ul> | <ul><li>**Vision Encoder & Linear Projection:** Processes the listing's interior/exterior photos (mapping directly to the multi-axis axial, coronal, and sagittal view inputs).</li><li>**Metadata (Patient Info):** Tabular `location` and text `description` are formatted into the system prompt.</li><li>**LLM + LoRA Output:** Estimates property value brackets (severity estimation) and generates selling points or warning highlights (precautionary recommendations).</li></ul> |
| **DeepFashion Multimodal**<br>[Marqo/deepfashion-multimodal](https://huggingface.co/datasets/Marqo/deepfashion-multimodal) | E-Commerce / Fashion | <ul><li>`image` (PIL Image)</li><li>`description` / `captions` (Text describing colors, textures, cuts, and categories)</li><li>`metadata` (Attribute category labels)</li></ul> | <ul><li>**Vision Encoder & Linear Projection:** Encodes clothing photos from different catalog views.</li><li>**Metadata (Patient Info):** The model merges the tabular category/material metadata into a single string.</li><li>**LLM + LoRA Output:** Generates rich product catalog details or performs structural attribute labeling (classification task).</li></ul> |
| **OBELICS**<br>[HuggingFaceM4/OBELICS](https://huggingface.co/datasets/HuggingFaceM4/OBELICS) | General Web Documents | <ul><li>`images` (List of image files/URLs)</li><li>`texts` (List of text segments)</li><li>`metadata` (Source webpage data)</li></ul> | <ul><li>**Vision Encoder & Linear Projection:** Handles sequences of images in interleaved positions, mimicking multi-image patient inputs.</li><li>**Metadata (Patient Info):** The text segments surrounding the images serve as the prompt context.</li><li>**LLM + LoRA Output:** Predicts the subsequent text tokens based on sequential visual-textual context (mimicking multimodal context-aware decision generation).</li></ul> |
| **Multimodal Open R1**<br>[lmms-lab/multimodal-open-r1-8k-verified](https://huggingface.co/datasets/lmms-lab/multimodal-open-r1-8k-verified) | Mathematics & Geometry Reasoning | <ul><li>`image` (Problem diagram)</li><li>`problem` (Text question)</li><li>`reasoning` / `thinking_process` (Text string enclosed in `<think>...</think>` tags)</li><li>`answer` (Final target choice/number)</li></ul> | <ul><li>**Vision Encoder & Linear Projection:** Ingests the problem diagram or visual context.</li><li>**Metadata (Patient Info):** The `problem` text functions as the clinical prompt.</li><li>**LLM + LoRA Output:** Instead of direct classification, the model generates an explicit reasoning trace first, followed by the final decision. This is highly applicable for clinical applications where visual diagnosis requires explanatory intermediate thinking before yielding a final classification.</li></ul> |
| **Docmatix**<br>[HuggingFaceM4/Docmatix](https://huggingface.co/datasets/HuggingFaceM4/Docmatix) | Document Visual QA (DocVQA) | <ul><li>`images` (List of multi-page document images)</li><li>`texts` (Conversational list format with `user` queries, `assistant` responses, and `source` markers)</li></ul> | <ul><li>**Vision Encoder & Linear Projection:** Maps multiple document pages (replacing 3D MRI cross-sections) to visual tokens.</li><li>**Metadata (Patient Info):** The `user` queries act as user instructions or system constraints.</li><li>**LLM + LoRA Output:** Generates structured textual extractions or QA validations (assistant responses) directly conditioned on visual document evidence.</li></ul> |
